D2 — SSID Architecture

Max 4 SSIDs per radio. Each SSID adds beacon overhead (~2% airtime per SSID). Beyond 4, overhead degrades throughput for all clients.

Horizon 2-SSID Model

SSIDSecurityVLANPurpose
Horizon-CorpWPA2/WPA3 Transition20Staff, trusted devices
Horizon-GuestOWE (Enhanced Open)30Visitors, BYOD

OWE Transition Mode

OWE encrypts open networks without a password. Legacy devices that don't support OWE need a fallback.

# How OWE Transition Mode works:
Horizon-Guest (OWE)      ← visible to OWE-capable clients → encrypted
Horizon-Guest-hidden     ← hidden Open SSID → legacy devices fall back here
                            Both are on same VLAN, same firewall rules
XIQ gotcha: OWE Transition Mode is a toggle inside the Guest SSID profile — not a separate SSID you create. If you create the hidden SSID manually, XIQ creates duplicate BSSIDs. Use the toggle only.

D3 — Channel Planning

2.4 GHz — The Rules

Only channels 1, 6, 11 — never use overlapping channels (2–5, 7–10). Never deploy 40 MHz on 2.4 GHz.

5 GHz UNII Bands

BandChannelsDFS?Recommendation
UNII-136, 40, 44, 48NoFirst choice — always available
UNII-2A52–64YesUse only if UNII-1/3 exhausted
UNII-2C100–140YesUse only if UNII-1/3 exhausted
UNII-3149, 153, 157, 161, 165NoFirst choice alongside UNII-1

6 GHz — Clean Spectrum WiFi 7

59 non-overlapping 20 MHz channels. No legacy devices, no DFS, no microwave. Clients use PSC (Preferred Scanning Channels) as entry points. Requires WiFi 6E or WiFi 7 AP (AP5010/AP5020 — not AP3000).

WiFi 7 Multi-Link Operation (MLO) WiFi 7

MLO allows a single client to maintain simultaneous links on 2.4 + 5 + 6 GHz bands at the same time. This changes roaming from a "switch AP" event to a "switch link" event — sub-millisecond, transparent to applications.

Roaming latency<20ms (802.11r)<1ms (link failover)
Max channel width160 MHz320 MHz (6 GHz only)
Modulation1024-QAM4096-QAM
Theoretical max9.6 Gbps46 Gbps

WPA3 SAE vs WPA2 PSK

PropertyWPA2-PSKWPA3-SAE
Key exchange4-way handshake (capturable)Dragonfly (mutual, not capturable)
Offline dictionary attackVulnerable (PMKID capture)Not possible
Forward secrecyNoYes — past traffic safe if key later compromised
PMF requirementOptionalMandatory
PMF (802.11w / Protected Management Frames) encrypts deauth and disassoc frames. Without PMF, an attacker can forge deauth frames to disconnect clients (deauth attack). WPA3 mandates PMF. For WPA2 deployments, set PMF to Optional (not Required) for maximum compatibility.

Trust Boundary

# Trust DSCP from Corp clients (managed devices mark correctly)
Phone marks DSCP EF → AP maps to AC_VO → 802.1p CoS 6 → preserved across Fabric

# Do NOT trust DSCP from Guest
Guest DSCP → reset to CS0 (best effort) at AP ingress
# Prevents Guest from marking all traffic AC_VO to steal priority

VOSS QoS Advantage

In VOSS SPB Fabric, 802.1p CoS is carried inside the MAC-in-MAC tunnel header across the NNI. QoS set at the UNI edge (Port 3) is preserved end-to-end through the Fabric without re-marking at each hop — a key advantage over VXLAN which strips CoS markings. VOSS

C4 — Guest Isolation

Two layers required: XIQ firewall rules stop L3 Guest→Corp routing. L2 station isolation stops Guest→Guest traffic at the AP level. Firewall rules alone don't stop same-VLAN L2 traffic between Guest clients.

XIQ Firewall Rules (applied in order)

RuleActionSourceDestination
1DENY10.30.0.0/24 (Guest)10.10.0.0/24 (Corp-Mgmt)
2DENY10.30.0.0/24 (Guest)10.20.0.0/24 (Corp-Users)
3PERMIT10.30.0.0/24 (Guest)Any (internet)

VOSS ACL — Must Be on Both Switches

Anycast posture change: In a VOSS Fabric with Anycast Gateway, SW2 routes locally — it never sends Guest traffic to SW1. If the ACL is only on SW1, Guest-to-Corp traffic from SW2 bypasses it entirely.
# Apply on BOTH SW1 and SW2 (VOSS Fabric Engine):
ip filter acl GUEST_ISOLATION
  deny ip 10.30.0.0/24 10.10.0.0/24
  deny ip 10.30.0.0/24 10.20.0.0/24
  permit ip any any
  exit
vlan filter ip GUEST_ISOLATION 30 in

C5 — Fabric Attach VOSS

FA Negotiation Flow

1. AP boots → gets DHCP on management VLAN (untagged)
2. AP sends LLDP frames with FA TLVs requesting I-SIDs
3. Switch receives: show fa neighbor → AP3000 MAC appears
4. Switch grants I-SIDs dynamically on that port
5. AP gets Corp + Guest VLANs without manual trunk config
6. Verify: show fa assignment → Port 3: I-SIDs Active + Dynamic

Switch CLI (VOSS)

# Enable FA globally (once)
fa enable

# Enable on AP port
interface gigabitEthernet 1/3
  auto-sense enable
  fa enable
  exit

# Verify
show fa neighbor          # AP3000 MAC on port 3
show fa assignment        # Port 3: 100020 + 100030 → Active, Dynamic
If show fa neighbor is empty: Check LLDP is not blocked on the switch port. Check auto-sense enable is configured. Check cable is seated.

P1 — AP Placement

Coverage Targets

EnvironmentCoverage RadiusHeight
Open office15–20m2.4–3m ceiling
Classroom / meeting rooms10–15m2.4m ceiling
Warehouse25–30m5–8m, angled down
Corridor20–25m (directional)End of corridor, angled

Placement Rules

  • Height: 2.4–3.0m ceiling is optimal. Below 2m = too close to clients, poor radiation pattern. Above 4m = 5 GHz coverage holes.
  • Cell overlap: 15–20% overlap between adjacent APs — prevents coverage holes without excessive co-channel interference.
  • Avoid: Lift shafts, metal walls, above suspended ceilings with dense concrete, near microwave ovens.
  • High density: Reduce TX power and use smaller cells rather than fewer high-power APs. Dense deployments are SNR problems, not signal strength problems.

P4 — Deployment Checklist

Phase 1 — Design Sign-Off

  • SSID count ≤ 4 per radio
  • Each SSID maps to exactly one VLAN
  • I-SIDs calculated: VLAN + 100,000
  • Security mode confirmed: WPA2/WPA3-Transition + OWE-Transition
  • Guest isolation design: XIQ firewall + L2 station isolation

Phase 2 — Switch Configuration

  • Karl Rule applied: enable ipforwarding vlan Default (EXOS)
  • DHCP enabled on ports 3, 5, 10 (port 10 = SW2 reach-back)
  • VOSS: IS-IS adjacency UP — show isis adj
  • VOSS: I-SIDs show "Both" — show isis spbm i-sid
  • VOSS: Guest ACL on both switches

Phase 3 — XIQ

  • IP Objects created (prerequisite for firewall rules)
  • Corp PSK Key Value populated
  • OWE Transition Mode toggle ON for Guest SSID
  • Complete Update pushed (not Partial)
  • AP Config Status: "In Sync"

Phase 4 — Post-Deployment Verification

  • Corp client: IP in 10.20.0.0/24, ping gateway, ping 8.8.8.8
  • Guest client: IP in 10.30.0.0/24, ping internet, CANNOT ping Corp
  • Guest → Guest ping FAILS (L2 station isolation)
  • Roaming test: walk client between APs, SLA tab latency <20ms
  • XIQ Network 360: all links green

T2 — Roaming Issues

Sticky Client

A client that refuses to roam despite a closer AP having better signal. Root cause: 802.11 gives roaming control to the client — the AP cannot force disconnection without 802.11v.

RSSI at Current APExpected Behaviour
≥ -65 dBmStay on current AP
-65 to -70 dBmAP considers 802.11v nudge
-70 to -75 dBmClient should roam if better AP visible
< -75 dBmMust roam — below usable threshold

Roaming Issues Quick Reference

SymptomCauseFix

WiFi 7 MLO roaming: With MLO, a "roam" becomes a link state change — one radio link degrades, another takes over. Sub-millisecond, transparent to the application. Traditional 802.11r is largely superseded for MLO clients. WiFi 7

T3 — RF Interference

SNR = Signal − Noise Floor

Normal noise floor: -95 dBm
RSSI: -68 dBm → SNR = 27 dB  ← Good
RSSI: -68 dBm, noise floor -80 dBm → SNR = 12 dB  ← Interference present

Interference Types

TypeSourceBandFix
Co-channel (CCI)Overlapping APs on same channel2.4 + 5 GHzFix channel plan; reduce TX power
Adjacent-channelAPs on overlapping channels (e.g., 1+2)2.4 GHz worst1/6/11 only — never adjacent channels
Non-WiFiMicrowave, BT, baby monitors2.4 GHz mostlyBand steering to 5 GHz; deploy 6 GHz AP
DFS radarRadar systemUNII-2A/2CMove to UNII-1/UNII-3

BSS Coloring (WiFi 6) WiFi 6

Tags each BSS with a 6-bit color (1–63). Frames from a different-color BSS are treated as background noise if below threshold — allows spatial reuse of the same channel in dense deployments without explicit coordination.

6 GHz: Zero Non-WiFi Interference WiFi 7

6 GHz has no microwave ovens, no Bluetooth, no legacy WiFi, no DFS radar. The single strongest argument for deploying WiFi 6E (AP5010) or WiFi 7 (AP5020) in high-density environments.
